🚀 nicemso note 실행 가이드 (소스코드 버전)

📂 프로젝트 폴더: 소스코드가 들어있는 폴더를 먼저 찾으세요

🎯 방법 1: 쉘 스크립트 실행 (가장 간단)

✨ 추천 방법! 한 번의 더블클릭으로 실행 가능
1단계: Finder에서 프로젝트 폴더 열기
2단계: run.sh 파일을 더블클릭
3단계: "터미널에서 열기" 선택
💡 참고: 처음 실행 시 보안 경고가 나타날 수 있습니다. "시스템 환경설정 → 보안 및 개인 정보 보호"에서 허용해주세요.

🖥️ 방법 2: 터미널 명령어

1단계: 터미널(Terminal) 열기
Spotlight(Cmd+Space) → "터미널" 검색
2단계: 프로젝트 폴더로 이동
cd /프로젝트/폴더/경로
💡 팁: Finder에서 폴더를 터미널로 드래그하면 경로가 자동 입력됩니다
3단계: 앱 실행
npm start

🔧 방법 3: VS Code에서 실행

1단계: VS Code로 프로젝트 폴더 열기
2단계: 터미널 패널 열기 Ctrl + `
3단계: 명령어 실행
npm start

📋 최초 설정 (한 번만 필요)

⚠️ 중요: 다른 사람이 소스코드를 받았을 때 최초 1회 실행 필요
1단계: Node.js 설치 확인
node --version
npm --version
버전이 표시되지 않으면 Node.js 공식 사이트에서 설치
2단계: 의존성 설치
npm install
이 명령어는 프로젝트에 필요한 라이브러리들을 설치합니다 (최초 1회만)
3단계: 앱 실행
npm start

🎮 단축키 안내

앱이 실행되면 다음 단축키들을 사용할 수 있습니다:

  • Command + E (또는 E): UI 수정 로직 실행
  • Command + S: 저장 버튼 클릭
  • Command + R: 웹뷰 새로고침
  • Enter (주소창에서): 페이지 이동

🔄 소스코드 수정 후

1단계: 실행 중인 앱 종료 Cmd + Q
2단계: 소스코드 수정 (index.html, main.js 등)
3단계: 다시 실행
npm start
✅ 장점: 빌드 과정 없이 즉시 변경사항 확인 가능!

🚨 문제 해결

npm start가 안 될 때:
npm install
npm start
포트 에러가 날 때:
다른 터미널에서 실행 중인 앱이 있는지 확인하고 종료
일렉트론이 안 열릴 때:
npm install electron --save-dev
npm start

📤 소스코드 공유 시 포함할 파일들

필수 파일들:
  • package.json - 프로젝트 설정
  • main.js - 일렉트론 메인 프로세스
  • index.html - UI 및 로직
  • run.sh - 실행 스크립트
  • README.md - 설명서 (선택사항)
제외할 폴더들:
  • node_modules/ - 자동 생성됨
  • dist/ - 빌드 결과물
🎉 완료! 이제 소스코드만 공유해도 누구나 쉽게 실행할 수 있습니다.
